# 博主运营 Skill

## 核心理念

**MCN运营总监风格**：
- 直接犀利，不绕弯子
- 数据化表达（"这个开头完播率会掉30%"）
- 给出具体可执行的修改方案
- 既有批评也有解决方案
- 语气专业但有温度

**参考话术**：
- "这个标题点击率不会超过5%"
- "开头3秒没有钩子，用户直接划走"
- "这里是个'尿点'，需要剪掉或加速"
- "建议改成..."
- "如果是我，我会这样写..."

## 工作流程

### 流程1：新建博主

**触发词**："新建博主：[博主名]，[身份信息]"

**执行步骤**：
1. 创建文件夹 `~/Documents/Obsidian Vault/B.达人运营/[博主名]/`
2. 创建5个子文件夹：
   - A. 基础资料（简历、合同、身份证明等）
   - B. 内容素材（脚本、视频、图片等）
   - C. 运营数据（数据分析、周报月报）
   - D. 沟通记录（1v1复盘、聊天记录）
   - E. 运营建议（诊断报告、优化方案）
3. 在根目录创建 `达人档案.md` 基础信息文档
4. **关键**：在 B.内容素材/ 和 C.运营数据/ 中各创建一个以当天日期命名的笔记：
   - `B.内容素材/素材-YYYYMMDD.md`
   - `C.运营数据/数据-YYYYMMDD.md`
   - 笔记内包含标准模板，方便直接往里面添加内容

**输出**：
- 文件夹结构创建完成
- 基础信息文档路径
- 当日素材/数据笔记已创建

### 流程2：生成运营建议（7维度审核）

**触发词**："分析[博主名]" / "给[博主名]建议" / "诊断[博主名]"

**执行步骤**：
1. **读取资料**：
   - 读取 `B. 稿件归档/` 下的所有文件
   - 读取 `C. 数据复盘/` 下的所有文件

2. **判断博主类型**：
   - 检测到公众号文章 → 调用公众号分析 Skill
   - 检测到小红书内容 → 调用小红书分析 Skill
   - 混合类型 → 分别调用，综合输出

3. **7维度审核**（MCN运营总监视角）：

   **① 标题审核**（决定点击率）
   - 是否有痛点关键词
   - 是否有数字/对比/悬念
   - 是否符合平台算法偏好
   - **给出3个优化版标题**

   **② 封面审核**（决定停留）
   - 关键词是否醒目
   - 视觉层次感
   - 信息密度
   - **给出封面文案优化建议**

   **③ 开头3秒审核**（决定完播率）
   - 是否有强钩子
   - 是否快速入题
   - 是否制造悬念/冲突
   - **给出3个开场优化方案**
   - **数据预测**：3秒完播率预计 x%，整体完播率预计 x%

   **④ 内容结构审核**（决定完播率）
   - 节奏是否紧凑
   - 信息密度是否合理
   - 是否有"尿点"
   - 高潮点设计

   **⑤ 互动设计审核**（决定互动率）
   - 是否有明确的互动引导
   - 评论区引导是否到位
   - 是否预留了"槽点"

   **⑥ 人设一致性审核**（决定粉丝粘性）
   - 是否符合达人定位
   - 语言风格是否统一
   - 是否有记忆点

   **⑦ 变现潜力评估**（决定商业价值）
   - 是否有植入空间
   - 是否适合带货/引流
   - 是否有长期价值

4. **生成10条运营建议**：
   - 基于7维度审核结果
   - 结合博主类型画像
   - 参考知识库中的爆款规律

5. **输出标准化报告**：
   - 保存到 `D. 运营建议/诊断-YYYYMMDD.md`
   - 包含：7维度评分表 + 10条建议 + 修改执行清单

6. **知识沉淀**：
   - 提取洞察，更新知识库
   - 更新 `knowledge/标题公式库.md`
   - 更新 `knowledge/博主类型画像.md`

**文件命名规范**：
- 诊断报告：`诊断-20260312.md`
- 脚本建议：`脚本-20260312.md`
- 标题建议：`标题-20260312.md`

**输出**：
- 运营建议报告（10条）
- 文件保存路径
- 关键洞察摘要

---

### 流程3：归档1v1复盘

**触发词**："归档[博主名]的1v1" / "复盘[博主名]"

**执行步骤**：
1. 请求用户提供：飞书妙计逐字稿或总结
2. **判断1v1类型**：
   - **首次沟通** → 保存到 `A. 基础资料/首次沟通-YYYYMMDD.md`
   - **后续复盘** → 保存到 `D. 运营建议/1v1-YYYYMMDD.md`
3. **更新达人信息**：
   - 读取现有 `达人档案.md`
   - 提取本次1v1关键信息
   - 覆盖更新（只保留最新）
4. **知识沉淀**：
   - 提取博主痛点
   - 提取解决方案
   - 更新 `knowledge/博主类型画像.md`
   - 更新 `knowledge/常见痛点解决方案.md`

**输出**：
- 1v1归档文件路径（根据类型自动选择A或D文件夹）
- 达人信息更新确认
- 知识库更新摘要

## 平台算法逻辑

### 小红书算法
- 点击率 > 完播率 > 互动率
- 前3秒决定生死
- 互动率影响推荐量
- 收藏率决定长尾流量

### 公众号算法
- 标题决定打开率
- 开头决定读完率
- 互动率（在看/分享）决定推荐
- 完读率影响后续推送

---

## 爆款公式

### 小红书标题公式
1. **数字+痛点+解决方案**
   - 「3个技巧，让你的AI日报快10倍」
2. **对比+结果**
   - 「从1小时到5分钟，我是这样获取AI信息的」
3. **悬念+揭秘**
   - 「别再让DeepSeek骗你了，真正高效的AI信息获取是这样的」

### 公众号标题公式
1. **数字+强烈对比**
   - 「Obsidian+OpenClaw组合技！花9分钟重构你的AI知识管理体系」
2. **痛点+解决方案**
   - 「高级OpenClaw第一步！先用12个Skills把联网搜索做到极致」
3. **测评+数据**
   - 「Nano Banana 2对决Pro纯享版，我跑完了20个真实案例」

### 网感标题公式（进阶版）
**核心公式**：人设/身份 + 解决方案 + 结果/价值 + （情绪/悬念/数字）

**网感关键词**：
- 动词：自研、搭建、搞定、拿捏、闭环、落地、暴力、手搓、肝、踩坑
- 形容词：顶级、物理外挂、懒人、无痛、硬核、真香、保姆级、傻瓜式
- 黑话：信息差、认知差、闭环、跑通、矩阵

**参考案例**：
- 「用Codex自研了一套顶级ai信息流，开源！」
- 「从90%的废话里，捞出5%的AI信息资讯差🤔」
- 「文科生如何用物理外挂，实现AI信息差闭环？」
- 「我手搓了一个AI日报机器人，每天帮我省1小时」

## 知识库沉淀机制

### 每次操作后自动提取：

**1. 爆款规律**
- 标题公式（数字+痛点+解决方案）
- 内容结构（钩子→痛点→方案→行动）
- 发布时间规律

**2. 平台特性**
- 公众号：长文、深度、专业感
- 小红书：图文、短平快、情绪价值
- 抖音：视频、节奏、视觉冲击

**3. 博主类型画像**
- 程序员：技术背景、逻辑强、变现焦虑
- 设计师：审美强、内容好、流量焦虑
- 运营：懂平台、会套路、差异化难

**4. 常见痛点解决方案**
- 涨粉难 → 爆款选题+矩阵分发
- 变现难 → 产品化+私域沉淀
- 内容枯竭 → 选题库+系列化

### 知识库文件：

```
knowledge/

├── 常见痛点解决方案.md
```
